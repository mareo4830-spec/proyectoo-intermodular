Ventajas frente a soluciones tradicionales

  • Coste de Implantación Cercano a Cero: Hardware asequible y software 100% Open Source.
  • Bajo Impacto Energético: Una Raspberry Pi consume apenas 3-5W frente a los 200W de un servidor rack convencional.
  • Resiliencia: El uso de contenedores Docker y paneles estáticos garantiza que el sistema no se colgará por falta de memoria.
  • Privacidad de Datos: Todo el análisis de tráfico se procesa de forma local en la empresa.

El Reto Técnico (Inconvenientes mitigados)

  • Limitaciones de Arquitectura (32 bits): El reto: El software moderno a veces resulta incompatible con Raspbian de 32 bits. Solución: Pivotamos hacia herramientas agnósticas (Homer, Darkstat, Fail2Ban) logrando un rendimiento superior.
  • Curva de Aprendizaje: El reto: La gestión de servidores Linux intimida. Solución: Hemos centralizado todo en un dashboard visual intuitivo y entregamos la red configurada en modo "plug-and-play".

Metodología Agile

El proyecto se gestionó organizando el trabajo en Sprints semanales usando Trello, permitiendo una adaptación rápida ante los imprevistos de la arquitectura ARM.

  1. Análisis de viabilidad del hardware.
  2. Despliegue de red y motor de contenedores Docker.
  3. Pivotaje estratégico hacia herramientas ultraligeras.
  4. Pruebas de estrés y configuración de seguridad activa.
